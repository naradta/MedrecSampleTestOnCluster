Introduction

This document describes about how to deploy medrec sample application in WebLogic cluster environment using Oracle Database.

 

 

Pre-Requisites

 

WLS Cluster Domain Setup

Install/setup WebLogic Server (WLS) 12.2.1.3 Generic installer and Configure domain with cluster environment.

 

Ref link for Downloads:

https://www.oracle.com/middleware/technologies/weblogic-server-installers-downloads.html

Oracle DB Setup

Install/Setup Oracle Express Edition (XE)/Enterprise Edition (EE) database to host medrec samples data. For samples development/testing 12c Container Database is used.

 

Ref link for DB loads:

https://www.oracle.com/database/technologies/oracle-database-software-downloads.html

 

SQLPlus Client Setup

Install/Setup Oracle SQLPlus Client in the host where you are executing samples script (In your WLS Admin Server Host). This software is to connect to oracle database This step is required only if all scripts wants to be executed at the host where WLS Admin Server is running.

 

Ref link for SQLPlus Client Download:

https://www.oracle.com/database/technologies/instant-client/linux-x86-64-downloads.html

 

 

Medrec App deployment Samples Setup Download

Git clone from https://github.com/naradta/MedrecSampleTestOnCluster

 

Configurations for DB and WLS:

1.     Navigate to target directory and follow below steps to update configuration values in property files.

cd <DOWNLOAD_DIR>

2.     Follow below steps to change DB configurations

Navigate to config directory

cd <DOWNLOAD_DIR>/config

 

Edit db_config.properties and update the configuration values appropriately.

Property Name

Remarks

DBA_USERNAME

DB Admin username. This is used to create merec DB user. Default value is SYSTEM

Ex: DBA_USERNAME=SYSTEM

DBA_PASSWORD

DB Admin Password

Ex: DBA_PASSWORD=oracle11

CONTAINER_DB

Represents Oracle DB is container or non-container DB. Possible values are YES/NO. Default value is NO

Ex: CONTAINER_DB=NO

PDB_NAME

Pluggable DB name. You need to provide this value when CONTAINER_DB value is set to YES. Default value is NA

 

Ex: PDB_NAME=NA

DB_USERNAME

Oracle Database User name used to host medrec application data.

Default value is medrec

 

Ex: DB_USERNAME=medrec

DB_PASSWORD

Oracle Database User Password. Default value is medrec

 

Ex: DB_PASSWORD=medrec

DB_HOSTNAME

Oracle Database Host name

 

Ex: DB_HOSTNAME=myhost.domain.com

DB_PORT

Database Port. Default value is 1521

 

Ex: DB_PORT=1521

DB_SID

Oracle Database SID or Service name. If both are different, provide service name of the DB

 

Ex: DB_SID=pdb1

DB_NAME

Oracle Database Name

 

Ex: DB_NAME=orcl

ORACLE_HOME

Oracle Home path of the Database

 

Ex: /u01/app/db/client_12c

SQLPLUS

Complete path for SQLPLUS. Usually it will be either under ORACLE_HOME or ORACLE_HOME/lib

 

Ex: /u01/app/db/client_12c/sqlplus

 

 

 

3.     Follow below steps to change WLS configurations

Navigate to config directory

$cd <TARGET_DIR>/config

 

Open wls_config.properties and update only below property values

Property Name

Remarks

WLS_USERNAME

Weblogic Admin User Name. Default value is weblogic.

Ex: WLS_USERNAME=weblogic

 

WLS_PASSWORD

Weblogic Admin User Password

Ex: WLS_PASSWORD=welcome1

 

WLS_ADMINHOST

Weblogic Admin Server Host Name

Ex: WLS_ADMINHOST=myhost.domain.com

WLS_ADMINPORT

Weblogic Admin Server Port. Default value is 7001.

Ex: WLS_ADMINPORT=7001

WLS_CLUSTERNAME

Target Cluster Name where we are deploying sample medrec App.

Ex: WLS_CLUSTERNAME=Cluster-1

WLS_ORACLE_HOME

Weblogic oracle home path

Ex: WLS_ORACLE_HOME=/u01/app/mw/oracle_home

WLS_DOMAIN_HOME

Weblogic domain home path

Ex: WLS_DOMAIN_HOME=/u01/app/mw/wls_domain

WLS_DOMAIN_NAME

Weblogic domain name

WLS_DOMAIN_NAME=wls_domain

FILE_STORE_PATH

Weblogic Managed Server Persistent File Store path for JMS Server.

Ex: FILE_STORE_PATH=/u01/app/mw/filestore

Configuring Oracle DB and Deploying App:

Scripts can be used with one of the following ways.

 

When SQL Client is configured in WLS admin host

This step depends on SQLPlus Client Setup

Follow below steps to setup medrec sample application.

$cd <DOWNLOAD_DIR>/scripts

$./setup.sh

 

Following activates are performed when setup.sh is executed

Validate DB connectivity and execute SQL script (If SQL Client is configured)

Setup datasource for oracle database

Setup Persistent File Store, JMS Server

Sub Deployment, Connection Factory and Queue

Setup Mail Session

Setup WLDF for medrec App

Import medrec app SQL data to oracle database

Deploy medrec sample apps to cluster and startup the applications.

 

When SQL Client is not configured in WLS admin host

In case sqlplus client is not configured

 

git clone https://github.com/naradta/MedrecSampleTestOnCluster

at DB server and host where WLS Admin server is running.

 

At DB Server

 

Update DB configuration file and then directly execute setupDB.sh to insert all DB changes.

        cd <DOWNLOAD_DIR>

        Edit config/db_config.properties

        Execute setupDB.sh

Or Manually execute below scripts as DBA user using sqlplus.

 

For Container DB:

Execute below script as sys dba user to create medrec user id

<DOWNLOAD_DIR>/scripts/cdb/configCDBUserManually.sql

 

Note:You need to change PDB_NAME,MEDREC_DB_USER_NAME, MEDREC_DB_USER_PASSWORD

 

Execute below script as medrec DB user to create tables

<DOWNLOAD_DIR>/scripts/cdb/sqlScriptsCDBManually.sql

Note:You need to change PDB_NAME, MEDREC_DB_USER_NAME.

 

 

For Non Container DB

Execute below script as sys dba user to create medrec user id

<DOWNLOAD_DIR>/scripts/ncdb/configNCDBUserManually.sql

 

Note:You need to change MEDREC_DB_USER_NAME,MEDREC_DB_USER_PASSWORD

 

Execute below script as medrec DB user to create tables

<DOWNLOAD_DIR>/scripts/ncdb/sqlScriptsNCDB.sql

 

At WLS Admin Server

        cd <TARGET_DIR>

        Execute setupWLS.sh

DB Actions performed

 

1.     When<DOWNLOAD_DIR>/scripts/cdb/configCDBUserManually.sql or configNCDBUserManually.sql is executed, it will create a new DB schema user, and provide required grants to create tables and views.

 

2.     When <DOWNLOAD_DIR>/scripts/cdb/sqlScriptsCDBManually.sql or sqlScriptsNCDBManually.sql is executed, it will create following tables in DB and inserts some records in these tables.

DRUGS, PATIENTS, PHYSICIANS, RECORDS, PATIENTS_RECORDS, PRESCRIPTIONS, RECORDS_PRESCRIPTIONS, SEQUENCE

3.     When <DOWNLOAD_DIR>/setupDB.sh or setup.sh is executed, both actions are performed in DB end.

 

Configurations Added in WLS

 

1.     For target WLS Cluster, Migration policy is changed/updated to Consensus

 

2.     For one of the Migrable Target, Migration Policy is changed to "Auto-Migrate Exeactely-Once Services". You can check it by following the menu Clusters >> Migratable Targets

 

 

3.     DB Data Source is created with name "MedRecGlobalDataSourceXA"

 

4.     Persistent Store is created with name medrecFS

 

5.     Following JMS Resources are created

JMS Server: MedRecJMSServer

JMS Module: MedRec-jms

JMS Queue: PatientNotificationQueue

JMS Connection Factory: MedRecConnectionFactory

JMS Sub Deployment: PatientNotificationQueue

 

6.     Following Mail Session created

mail/MedRecMailSession

 

7.     Following Diagnostic Modules created

MedRecWLDF
